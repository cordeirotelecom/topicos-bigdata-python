O que é Apache Hadoop?

O Apache Hadoop é um framework open-source que revolucionou o mundo do Big Data, permitindo o processamento e armazenamento distribuído de grandes volumes de dados em clusters de computadores commodity. Desenvolvido por Doug Cutting e Mike Cafarella em 2006, baseado nos famosos papers do Google sobre MapReduce e Google File System.

Por que Hadoop é Revolucionário?

  • Escalabilidade Linear: Crescimento proporcional com adição de nodes
  • Tolerância a Falhas: Funciona mesmo com falhas de hardware
  • Hardware Commodity: Usa servidores padrão, não supercomputadores
  • Processamento Local: Move código para dados, otimizando transferências
  • Flexibilidade Total: Dados estruturados, semi-estruturados e não-estruturados

Evolução Histórica Completa

1
2003-2004: Papers Fundacionais do Google

Marco Inicial: Google publica os papers revolucionários que inspiraram o Hadoop:

  • "The Google File System" (SOSP 2003): Fundamentos do armazenamento distribuído
  • "MapReduce: Simplified Data Processing" (OSDI 2004): Modelo de programação paralela

Impacto: Demonstrou como processar petabytes de dados com hardware commodity, inspirando toda uma nova geração de sistemas distribuídos.

2
2005-2006: Nascimento do Hadoop

Criação: Doug Cutting (criador do Lucene) e Mike Cafarella desenvolvem o Hadoop como parte do projeto Nutch no Yahoo!, focando em indexação web distribuída.

Nome Curioso: "Hadoop" vem do nome do elefante de pelúcia amarelo do filho de Doug Cutting - daí o mascote elefante!

Objetivo Inicial: Competir com Google na indexação de bilhões de páginas web.

3
2008: Apache Foundation

Governança: Hadoop torna-se projeto top-level da Apache Software Foundation, ganhando estrutura formal de governança e comunidade global de desenvolvedores.

Versão 0.20: Primeira versão estável com APIs definidas e documentação completa.

Adoção Inicial: Yahoo! processa mais de 10 petabytes de dados em produção.

4
2009-2011: Expansão do Ecossistema

Surgimento de Ferramentas Complementares:

  • Apache Hive (2009): SQL para Hadoop - facilita análises com linguagem familiar
  • Apache HBase (2010): Banco NoSQL distribuído sobre HDFS
  • Apache Pig (2010): Linguagem de alto nível para transformações de dados
  • Apache Flume (2011): Coleta distribuída de logs em tempo real
  • Apache Oozie (2011): Orquestração de workflows complexos

Resultado: Hadoop evolui de framework de armazenamento para plataforma completa de Big Data.

5
2012: Revolução YARN (Hadoop 2.0)

Game Changer: Introdução do YARN (Yet Another Resource Negotiator), a maior evolução desde a criação do Hadoop.

Breaking Change Importante

Hadoop 2.0 introduz incompatibilidades com versão 1.x, mas oferece flexibilidade revolucionária.

Benefícios do YARN:

  • Separação do gerenciamento de recursos do processamento
  • Múltiplas aplicações simultâneas além do MapReduce
  • Melhor utilização de recursos do cluster
  • Suporte nativo para Spark, Storm e outras engines
6
2013-2017: Maturidade Empresarial

Adoção Enterprise: Hadoop atinge maturidade para uso empresarial crítico:

  • Apache Spark (2013): Engine de processamento in-memory 100x mais rápida
  • Apache Kafka (2014): Streaming distribuído para dados em tempo real
  • Security Enhancement: Kerberos, encryption, autorização granular
  • Cloud Integration: AWS EMR, Azure HDInsight, Google Dataproc
  • SQL-on-Hadoop: Impala, Presto, Drill para analytics interativo

Milestone: Facebook processa 300+ petabytes diariamente com Hadoop.

7
2018-2025: Era Cloud-Native

Adaptação Moderna: Hadoop evolui para realidades cloud e containers:

  • Kubernetes Integration: Deployers nativos para containers
  • Storage Separation: Compute e storage independentes (S3, Azure Data Lake)
  • Edge Computing: Processing distribuído em dispositivos IoT
  • AI/ML Native: TensorFlow, PyTorch integrados nativamente
  • Serverless Hadoop: Functions-as-a-Service para MapReduce

Futuro: Hadoop 4.0 promete compatibilidade total com arquiteturas serverless e edge.

O que é Apache Hadoop?

O Apache Hadoop é um framework open-source que revolucionou o processamento de Big Data, permitindo o armazenamento e processamento distribuído de datasets massivos em clusters de computadores commodity. Criado em 2006 por Doug Cutting e Mike Cafarella, baseado nos papers revolucionários do Google sobre MapReduce e Google File System.

Por que Hadoop é Revolucionário?

  • Escalabilidade Linear: Adicione nodes para crescer proporcionalmente
  • Tolerância a Falhas: Sistema continua funcionando mesmo com falhas de hardware
  • Custo-Efetivo: Usa hardware commodity em vez de supercomputadores
  • Flexibilidade: Processa dados estruturados, semi-estruturados e não estruturados
  • Processamento Local: Move código para dados, não dados para código

Evolução Histórica Detalhada

1

2003-2004: Fundações Teóricas

Google publica os papers revolucionários sobre Google File System (GFS) e MapReduce, estabelecendo os princípios fundamentais para processamento distribuído de big data em larga escala.

Papers Fundamentais:
• "The Google File System" (SOSP 2003)
• "MapReduce: Simplified Data Processing on Large Clusters" (OSDI 2004)
2

2005-2006: Nascimento do Hadoop

Doug Cutting (ex-Lucene) e Mike Cafarella criam o Hadoop como parte do projeto Nutch no Yahoo!, focando inicialmente em indexação web distribuída para competir com Google.

Marco Histórico: Nome "Hadoop" vem do elefante de pelúcia amarelo do filho de Doug Cutting.
3

2008: Apache Foundation

Hadoop torna-se projeto top-level da Apache Software Foundation, ganhando governança formal e comunidade global de desenvolvedores, estabelecendo as bases para crescimento exponencial.

4

2009-2011: Expansão do Ecossistema

Surgem projetos complementares que expandem drasticamente as capacidades do Hadoop:

  • Apache Hive (2009): Data warehouse com SQL-like queries
  • Apache HBase (2010): Banco NoSQL distribuído
  • Apache Pig (2010): Linguagem de alto nível para análise de dados
  • Apache Flume (2011): Coleta distribuída de logs
5

2012: Revolução YARN (Hadoop 2.0)

Introdução do YARN (Yet Another Resource Negotiator), separando gerenciamento de recursos do processamento, permitindo múltiplas aplicações simultâneas além do MapReduce.

Breaking Change: Hadoop 2.0 introduz incompatibilidades com 1.x, mas oferece muito mais flexibilidade.
6

2013-2017: Era da Maturidade

Hadoop atinge maturidade enterprise com:

  • Apache Spark: Engine de processamento in-memory
  • Apache Kafka: Streaming distribuído
  • Security Enhancements: Kerberos, encryption, autorização
  • Cloud Integration: AWS EMR, Azure HDInsight, Google Dataproc
7

2018-2025: Hadoop Moderno

Adaptação às novas realidades tecnológicas:

  • Kubernetes Integration: Containerização nativa
  • Cloud-Native: Storage separation (S3, Azure Data Lake)
  • Edge Computing: Processing distribuído em IoT
  • AI/ML Integration: TensorFlow, PyTorch sobre Hadoop

Instalação Profissional Completa

Guia completo para instalação do Apache Hadoop em diferentes ambientes, com configurações otimizadas para produção e desenvolvimento.

Instalação Profissional no Windows

Pré-requisitos Obrigatórios

  • Windows 10/11 Pro (versão 1903 ou superior)
  • Java 8 JDK (Oracle ou OpenJDK) - Exatamente versão 8
  • PowerShell 5.1+ executado como Administrador
  • 8GB+ RAM recomendado (mínimo 4GB)
  • 50GB+ espaço livre para dados, logs e aplicações
  • Windows Subsystem for Linux (WSL2) opcional mas recomendado
1

Configuração Completa do Java

Instale e configure o Java Development Kit 8 (obrigatório para compatibilidade):

PowerShell (Admin)
PS C:\> java -version
openjdk version "1.8.0_312" OpenJDK Runtime Environment (build 1.8.0_312-b07) OpenJDK 64-Bit Server VM (build 25.312-b07, mixed mode)

Configure as variáveis de ambiente no PowerShell como Administrador:

PowerShell
# Definir JAVA_HOME para sua instalação específica [Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Java\jdk1.8.0_312", "Machine") # Adicionar Java ao PATH do sistema $currentPath = [Environment]::GetEnvironmentVariable("PATH", "Machine") $javaPath = "%JAVA_HOME%\bin" $newPath = $currentPath + ";" + $javaPath [Environment]::SetEnvironmentVariable("PATH", $newPath, "Machine") # Verificar configuração (reinicie PowerShell) echo $env:JAVA_HOME java -version javac -version
✅ Verificação: Os comandos java -version e javac -version devem retornar a versão 1.8.x sem erros.
2

Download e Configuração do Hadoop

Baixe a versão LTS mais recente do Apache Hadoop com verificação de integridade:

PowerShell
# Criar estrutura de diretórios New-Item -ItemType Directory -Path "C:\hadoop" -Force New-Item -ItemType Directory -Path "C:\hadoop\data" -Force New-Item -ItemType Directory -Path "C:\hadoop\logs" -Force # Download Hadoop 3.3.6 LTS (versão estável atual) $hadoopUrl = "https://archive.apache.org/dist/hadoop/common/hadoop-3.3.6/hadoop-3.3.6.tar.gz" $hadoopFile = "C:\hadoop\hadoop-3.3.6.tar.gz" # Download com barra de progresso Invoke-WebRequest -Uri $hadoopUrl -OutFile $hadoopFile -UseBasicParsing # Verificar download Get-Item $hadoopFile | Select-Object Name, Length, LastWriteTime

Extração Manual

Use 7-Zip ou WinRAR para extrair o arquivo .tar.gz:

  1. Extrair o .gz primeiro: hadoop-3.3.6.tar.gz → hadoop-3.3.6.tar
