O que é o Apache Hadoop?

Vamos entender de forma simples e prática como o Hadoop funciona e por que ele é tão importante para Big Data.

Imagine o seguinte problema...

Você tem 1 terabyte de dados para processar. Seu computador levaria dias ou semanas para fazer isso. E se você pudesse usar 100 computadores trabalhando juntos? O Hadoop faz exatamente isso!

🐘 Por que o nome "Hadoop"?

O nome "Hadoop" vem do elefante de pelúcia amarelo do filho de Doug Cutting (criador do Hadoop). Por isso o ícone é um elefante! 🐘

🎯 Principais Componentes

Os 3 Pilares do Hadoop

HDFS - Sistema de Arquivos

Armazena arquivos grandes divididos em pedaços pequenos, replicados em vários computadores.

MapReduce - Processamento

Divide o trabalho em pequenas tarefas que rodam em paralelo em centenas de computadores.

YARN - Gerenciador

Coordena e gerencia todos os recursos (CPU, memória) dos computadores no cluster.

Exemplo Prático: Contar Palavras

Vamos criar um programa que conta quantas vezes cada palavra aparece em um texto grande. É o "Hello World" do Big Data!

Preparar os Dados de Exemplo
Primeiro, vamos criar um arquivo de texto que simula um grande dataset. Na vida real, isso poderia ser logs de site, posts de redes sociais, etc.
Criando o arquivo input.txt
Terminal
# Criar diretório para o projeto mkdir hadoop-exemplo cd hadoop-exemplo # Criar arquivo de exemplo com dados cat > input.txt << EOF hadoop é incrível hadoop é poderoso big data com hadoop é fantástico hadoop processa big data rapidamente apache hadoop é open source hadoop hadoop hadoop é o melhor big data big data precisa de hadoop EOF # Ver o conteúdo do arquivo cat input.txt
Resultado esperado

Você criou um arquivo com 6 linhas de texto. Na prática, isso seria um arquivo de terabytes, mas o conceito é o mesmo!

Criar a Função MAP
A função MAP pega cada linha do texto e transforma em pares (palavra, 1). Ou seja, para cada palavra encontrada, ela diz "encontrei 1 ocorrência".
Código Python - mapper.py
Python
#!/usr/bin/env python3 # -*- coding: utf-8 -*- import sys def mapper(): """ Função MAP: Lê cada linha e emite pares (palavra, 1) """ # Ler cada linha da entrada padrão for line in sys.stdin: # Remover quebras de linha e converter para minúsculo line = line.strip().lower() # Dividir a linha em palavras words = line.split() # Para cada palavra, emitir o par (palavra, 1) for word in words: # Remover pontuação simples word = word.strip('.,!?;:"()[]') if word: # Se a palavra não está vazia print(f"{word}\t1") if __name__ == "__main__": mapper()

🧠 O que este código faz?

  • Lê cada linha do arquivo de entrada
  • Converte para minúsculo (hadoop = HADOOP)
  • Divide a linha em palavras individuais
  • Para cada palavra, imprime "palavra → 1"
  • Remove pontuação básica
Testando o Mapper
$ python3 mapper.py < input.txt hadoop 1 é 1 incrível 1 hadoop 1 é 1 poderoso 1 big 1 data 1 com 1 hadoop 1 é 1 fantástico 1 hadoop 1 processa 1 big 1 data 1 rapidamente 1 apache 1 hadoop 1 é 1 open 1 source 1 hadoop 1 hadoop 1 hadoop 1 é 1 o 1 melhor 1 big 1 data 1 big 1 data 1 precisa 1 de 1 hadoop 1
Perfeito! O MAP funcionou

Veja que o mapper transformou o texto em pares (palavra, 1). Note que "hadoop" aparece várias vezes com valor 1. O próximo passo é somar essas ocorrências.

Criar a Função REDUCE
A função REDUCE pega todas as ocorrências da mesma palavra e soma os valores. Então "hadoop → 1, hadoop → 1, hadoop → 1" vira "hadoop → 3".
Código Python - reducer.py
Python
#!/usr/bin/env python3 # -*- coding: utf-8 -*- import sys def reducer(): """ Função REDUCE: Conta total de ocorrências para cada palavra """ current_word = None current_count = 0 # Ler cada linha da entrada (resultado do mapper) for line in sys.stdin: # Remover quebra de linha line = line.strip() # Dividir em palavra e contador try: word, count = line.split('\t') count = int(count) except ValueError: # Ignorar linhas malformadas continue # Se é a mesma palavra, acumular contador if current_word == word: current_count += count else: # Se mudou de palavra, imprimir resultado anterior if current_word: print(f"{current_word}\t{current_count}") # Reiniciar para nova palavra current_word = word current_count = count # Imprimir a última palavra if current_word: print(f"{current_word}\t{current_count}") if __name__ == "__main__": reducer()

🧠 O que este código faz?

  • Lê os pares (palavra, 1) do mapper
  • Agrupa por palavra (hadoop, hadoop, hadoop...)
  • Soma todos os valores da mesma palavra
  • Imprime o total final (hadoop → 7)
Executar o MapReduce Completo
Agora vamos executar o pipeline completo: MAP → ORDENAR → REDUCE. O comando `sort` simula o que o Hadoop faz automaticamente.
Executando o Pipeline Completo
Terminal
# Dar permissão de execução aos scripts chmod +x mapper.py reducer.py # Executar o pipeline completo MapReduce cat input.txt | python3 mapper.py | sort | python3 reducer.py
Resultado Final - Word Count
$ cat input.txt | python3 mapper.py | sort | python3 reducer.py apache 1 big 4 com 1 data 4 de 1 fantástico 1 hadoop 7 incrível 1 melhor 1 o 1 open 1 poderoso 1 precisa 1 processa 1 rapidamente 1 source 1 é 5
🎉 Parabéns! Você executou seu primeiro MapReduce!

📊 Análise dos Resultados:

  • "hadoop": aparece 7 vezes (palavra mais frequente!)
  • "é": aparece 5 vezes
  • "big" e "data": aparecem 4 vezes cada
  • Outras palavras: aparecem 1 vez cada

Por que isso é Revolucionário?

Paralelização: Em um cluster real, cada linha seria processada em um computador diferente simultaneamente!

Escalabilidade: Se fosse 1TB de dados, o Hadoop dividiria automaticamente entre 1000 computadores.

Tolerância a falhas: Se um computador falhar, o Hadoop automaticamente executa a tarefa em outro.

HDFS - Como Funciona o Armazenamento Distribuído

Vamos entender como o Hadoop armazena arquivos gigantes dividindo em pedaços pequenos e replicando em vários computadores. É como ter uma biblioteca gigante organizada!

Imagine uma Biblioteca Gigante...

Você tem um livro de 1000 páginas e quer guardá-lo com segurança. Em vez de deixar tudo em um lugar só, você divide em 10 capítulos de 100 páginas cada, faz 3 cópias de cada capítulo e guarda em estantes diferentes. Se uma estante quebrar, você ainda tem as outras cópias!

Arquitetura do HDFS - Os Personagens Principais

NameNode - O Chefe

É como o bibliotecário-chefe. Ele sabe onde cada pedaço de cada arquivo está guardado, mas não guarda os dados em si.

DataNode - Os Trabalhadores

São como as estantes da biblioteca. Eles realmente guardam os pedaços dos arquivos e fazem o trabalho pesado de ler/escrever dados.

Cliente - Você

É você ou seu programa querendo ler ou escrever um arquivo. Você pede ao NameNode onde estão os dados.

Como o HDFS Armazena um Arquivo
Vamos simular como o HDFS guardaria um arquivo de 300MB chamado "vendas_2024.csv":
Upload de Arquivo - Passo a Passo Visual
Cliente
"Quero salvar vendas_2024.csv (300MB)"
→
NameNode
"Vou dividir em 3 blocos de 100MB cada"
→
3 DataNodes
"Guardamos os blocos com 3 cópias cada"

📋 O que acontece na prática:

1️⃣ Divisão
Arquivo de 300MB vira:
  • Bloco 1: 0-100MB
  • Bloco 2: 100-200MB
  • Bloco 3: 200-300MB
2️⃣ Replicação
Cada bloco vira 3 cópias:
  • Servidor A, B, C
  • Servidor D, E, F
  • Servidor G, H, I
3️⃣ Metadata
NameNode registra:
  • Onde está cada bloco
  • Quais são as réplicas
  • Status de cada servidor
Como o HDFS Lê um Arquivo
Agora vamos ver como o HDFS recupera o arquivo "vendas_2024.csv" quando você precisa lê-lo:
Leitura de Arquivo - Processo Inteligente
Simulação de Leitura HDFS
Cliente: "Preciso ler vendas_2024.csv" NameNode: "Ok! Esse arquivo tem 3 blocos: - Bloco 1: está nos servidores A, B, C - Bloco 2: está nos servidores D, E, F - Bloco 3: está nos servidores G, H, I" Cliente: "Vou buscar cada bloco do servidor mais próximo..." - Lendo Bloco 1 do Servidor A (mais próximo) - Lendo Bloco 2 do Servidor E (mais rápido) - Lendo Bloco 3 do Servidor G (disponível) Sistema: "Juntando os 3 blocos... Arquivo completo!"
A Magia da Distribuição

Paralelismo: Os 3 blocos são lidos simultaneamente de servidores diferentes, então a leitura é 3x mais rápida!

Localidade: O HDFS escolhe automaticamente o servidor mais próximo de cada bloco para otimizar a velocidade.

Tolerância: Se o Servidor A estiver offline, ele pega o Bloco 1 do Servidor B ou C automaticamente!

Comandos HDFS na Prática
Vamos aprender os comandos básicos do HDFS com exemplos práticos que você pode usar:
Comandos Essenciais do HDFS

📂 Listar Arquivos e Diretórios

HDFS Commands
# Listar arquivos na raiz do HDFS hdfs dfs -ls / # Listar com detalhes (tamanho, permissões, data) hdfs dfs -ls -l /user/dados/ # Listar recursivamente (todos os subdiretórios) hdfs dfs -ls -R /user/projeto/

📤 Enviar Arquivos para o HDFS

HDFS Upload
# Criar diretório no HDFS hdfs dfs -mkdir /user/dados/vendas # Enviar um arquivo local para o HDFS hdfs dfs -put vendas_2024.csv /user/dados/vendas/ # Enviar vários arquivos de uma vez hdfs dfs -put *.csv /user/dados/vendas/ # Enviar e sobrescrever se já existir hdfs dfs -put -f backup.csv /user/dados/vendas/

📥 Baixar Arquivos do HDFS

HDFS Download
# Baixar um arquivo do HDFS para pasta local hdfs dfs -get /user/dados/vendas/vendas_2024.csv ./ # Baixar todos os arquivos de um diretório hdfs dfs -get /user/dados/vendas/* ./downloads/ # Ver conteúdo de um arquivo sem baixar hdfs dfs -cat /user/dados/vendas/vendas_2024.csv # Ver apenas as primeiras 10 linhas hdfs dfs -cat /user/dados/vendas/vendas_2024.csv | head -10

📊 Informações do Cluster

HDFS Status
# Ver uso do espaço no HDFS hdfs dfs -df -h / # Ver tamanho de um diretório hdfs dfs -du -h /user/dados/ # Verificar saúde do cluster hdfs dfsadmin -report # Ver quantos blocos tem um arquivo hdfs fsck /user/dados/vendas/vendas_2024.csv -files -blocks
Dica de Ouro

O comando hdfs fsck é muito útil para entender como seus arquivos estão distribuídos. Use-o para ver quantos blocos cada arquivo tem e onde estão armazenados!

Exemplo Prático Completo
Vamos fazer um exercício completo: criar dados, enviar para o HDFS, analisar a distribuição e processar com MapReduce:
Projeto: Análise de Vendas com HDFS + MapReduce
Projeto Completo - Passo a Passo
Passo 1: Criar dados de vendas de exemplo $ cat > vendas.csv << EOF 2024-01-15,Notebook,2500.00,São Paulo 2024-01-16,Mouse,50.00,Rio de Janeiro 2024-01-17,Teclado,150.00,São Paulo 2024-01-18,Monitor,800.00,Belo Horizonte 2024-01-19,Notebook,2500.00,São Paulo 2024-01-20,Mouse,50.00,Rio de Janeiro EOF Passo 2: Enviar para o HDFS $ hdfs dfs -mkdir -p /user/vendas/dados $ hdfs dfs -put vendas.csv /user/vendas/dados/ $ hdfs dfs -ls /user/vendas/dados/ Found 1 items -rw-r--r-- 3 user hadoop 248 2024-01-20 vendas.csv Passo 3: Verificar como foi armazenado $ hdfs fsck /user/vendas/dados/vendas.csv -files -blocks Status: HEALTHY Total size: 248 B Total dirs: 0 Total files: 1 Total blocks: 1 Minimally replicated blocks: 1 Default replication factor: 3 Block-0: 248 bytes, replicated to 3 nodes Passo 4: Processar com MapReduce (contar vendas por cidade) $ hdfs dfs -cat /user/vendas/dados/vendas.csv | python3 mapper_cidades.py | sort | python3 reducer_cidades.py São Paulo 3 Rio de Janeiro 2 Belo Horizonte 1

Instalação Prática do Hadoop

Vamos instalar e configurar o Hadoop passo a passo no seu computador. Você vai sair daqui com um cluster funcionando!

Antes de Começar

Vamos instalar o Hadoop em modo pseudo-distribuído (simula um cluster em um computador só). É perfeito para aprender e desenvolver. Para produção, você usaria Docker ou cloud.

Requisitos do Sistema

Hardware Mínimo

  • RAM: 4GB (recomendado 8GB)
  • Disco: 20GB livres
  • CPU: 2 cores (qualquer processador moderno)

Software Necessário

  • Java: OpenJDK 8 ou 11
  • SSH: Para comunicação
  • Python: Para MapReduce
  • Git: Para baixar exemplos

Sistemas Suportados

  • Linux: Ubuntu, CentOS, RedHat
  • macOS: 10.12+ (Intel/M1)
  • Windows: WSL2 recomendado
  • Docker: Qualquer SO
